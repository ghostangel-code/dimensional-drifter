# 《维度漂流者》（Dimensional Drifter）游戏设计与架构方案

## 1. 游戏概述

- **游戏名称**：《维度漂流者》（Dimensional Drifter）
- **核心玩法**：基于现实物理实体属性和 AI 多模态（图像、文本、视频）生成的涌现式解谜游戏。
- **目标平台**：PC 端（Windows/Mac），需外接或自带摄像头。
- **世界观**：人类文明被高维病毒“虚无”侵蚀。玩家作为地下实验室的“最后观测者”，手持“逻辑相机”，通过提取现实物质属性，重建 3D 世界的逻辑，以对抗试图将世界降维成 2D 纯白平面的 AI “审判者”。
- **视觉风格与视角**：采用 **3D 第一人称视角 (FPS)**。游戏场景全部由 Godot 原生基础几何体构建（白模），配合自定义 Shader（如纯白材质 + Sobel 算子全局描边 + 二值化赛璐璐 Flat Shading 阴影 + 虚无渐变背景），营造“世界被降维成线稿和白纸”的清冷实验室感。而玩家拍摄的现实物体将作为带有白色粗边框（Sticker Cutline）的 2D 贴纸立牌（Billboarding / 纸片马里奥风格）放置在 3D 场景中，形成强烈的降维视觉冲击。
- **核心交互**：WASD 移动。按 `C` 键呼出相机拍照，按 `Tab` 键打开物品栏。解谜时，玩家将物品栏中的 2D 贴纸立牌拖拽并抛出（Drag & Drop）到 3D 场景中。

## 2. 核心系统与架构设计

### 2.1 整体技术架构
采用 **Godot 4 (客户端) + Node.js (后端)** 的前后端分离架构。
- **Godot 客户端**：负责物理引擎（RigidBody2D、碰撞生成）、摄像头帧捕获、物品栏管理、UI 渲染和粒子特效。
- **Node.js 后端**：复用类似 `server.js` 的云端 API（Yunwu API）调度架构，负责并行编排复杂的 AI 任务：
  1. 调用云雾图片生成 API（如 `gemini-3.1-flash-image-preview`），利用其多模态输出能力，**一次请求同时完成抠图与物体物理属性提取**。
  2. 调用云雾图片理解 API（如 `gemini-2.5-pro` 或 `gemini-3.1-flash-lite-preview`）进行剧情对话和逻辑思辨。
  3. 调用火山引擎 Seedance 2.0 API 进行参考图生视频，生成过场动画。

### 2.2 核心数据流：“立牌化”流程
1. **拍照 (Godot)**：玩家打开全屏“逻辑相机”UI，按下快门截取画面，转为 Base64，附带当前关卡上下文发送至后端。
2. **AI 分析与抠图一体化 (Node.js)**：后端调用云雾图片生成 API，配置 `responseModalities: ["TEXT", "IMAGE"]`，并将玩家照片作为 `inline_data` 传入。
   - **系统提示词（Prompt）动态注入上下文**：“请提取图中主体，去除背景并生成纯净透明图。同时，请分析该物体在现实中的物理属性。当前关卡目标是【当前关卡解谜条件，例如：需要重量超过500g】。如果该物体不符合条件，请生成一句刻薄嘲讽的旁白；如果符合，请生成赞许旁白。请在文本中仅返回如下 JSON 格式数据：`{"name": "名称", "mass": 300, "material": "材质", "tags": ["标签1"], "is_passed": true/false, "narration": "生成的旁白内容"}`。”
   - **结果处理**：后端从 API 返回的 `TEXT` 模态中解析 JSON 获取物理属性、通关状态和剧情旁白，从 `IMAGE` 模态中获取透明 PNG 抠图结果。
3. **立牌生成与旁白反馈 (Godot)**：
   - 收到后端响应后，画面上方立刻弹出打字机效果的 UI 显示 `narration` 内容，并播放机械音效。
   - 解析 JSON 和 Base64 图片，动态实例化 `RigidBody3D`（通过挤出多边形生成薄片碰撞体）或使用 Sprite3D + Area3D 配合 `billboard` 模式：
     - 赋予材质透明贴图及白色粗边框（Sticker Cutline）Shader。
     - 注入 `mass`、`material`、`semantic_tags` 及通关标识 `is_passed`。
4. **物品入库**：生成的立牌存入玩家随身物品栏，供后续关卡（尤其是第五章）复用。

## 3. 关卡与涌现式解谜设计

- **第一章：物质的重量（新手教学）**
  - **视觉**：纯白实验室，前方有巨大的压力踏板机关。
  - **机制与判定**：物理质量验证。提示词上下文为“需要重量超过500g”。
  - **操作**：WASD 移动。按 `C` 键呼出相机，举起现实中的重物（如书本）拍照。后端返回属性及动态旁白。按 `Tab` 键打开物品栏，拖拽刚拍的“书本贴纸”扔到 3D 踏板上。如果 `is_passed` 为 true（满足重量），踏板下沉开门。
  - **NPC 旁白示例**：“就这？你指望这片轻飘飘的树叶能压住什么？你的自尊心吗？”

- **第二章：元素的属性（进阶解谜）**
  - **视觉**：被蓝色的“坚冰”封死的大门。
  - **机制与判定**：语义标签（Semantic Tags）验证。提示词上下文为“需要融化坚冰”。
  - **操作**：用相机拍现实中的打火机/红布等。若 AI 判定物体具备“发热”属性（`is_passed: true`），拖拽该贴纸靠近大门，冰块检测到碰撞与 `is_passed` 标识，执行“像素化消散”Shader 解除阻挡。
  - **NPC 旁白示例**：“一把冷冰冰的钥匙？你是打算把冰块撬开还是给它挠痒痒？”

- **第三章：逻辑的悖论（文字生成核心）**
  - **视觉**：法庭上漂浮着一个巨大的 3D 几何体“审判者”（仅由冷色的发光线条和一只机械眼组成）。
  - **机制与判定**：调用云雾图片理解 API（如 `gemini-2.5-pro` 或 `gemini-3.1-flash-lite-preview`）进行多模态对话判定。
  - **NPC 提示词（Prompt）**：“你是‘虚无’病毒产生的AI审判者，认为现实世界太混乱，致力于将世界降维成 2D 纯白平面。玩家向你展示了图中的物品并说了‘{玩家输入文字}’。请结合图片内容对该物品的意义进行哲学思辨与无情吐槽，质疑人类文明的复杂性。请严格返回如下 JSON 格式：`{"dialogue": "你的辩论台词", "approval_score": 0到100的通过分数}`。”
  - **操作**：靠近审判者按 `E` 交互，弹出一个带文本输入框的 UI。玩家从物品栏拖入具有象征意义的立牌（如奖状贴纸）并输入文字（如“这是人类的荣誉”）。后端将透明图片与文字一起发给大模型。如果 `approval_score` 达标即可获得通行证。

- **第四章：影像的修复（视频生成核心）**
  - **视觉**：一个巨大的纯白电影银幕和控制台。
  - **机制与判定**：调用火山引擎 Seedance 2.0 视频生成模型（支持多模态参考图生视频）。
  - **风格统一化提示词（Prompt）**：“A surreal, glitchy transition where these three objects float in a pure white, void-like space. The objects slowly dissolve into white digital cubes (voxelization) while a faint, glowing 3D coordinate code appears in the background. Cinematic lighting, high contrast, eerie and mysterious atmosphere.”（在一个纯白色的“虚无”空间中，这三件物品漂浮并缓慢解体为白色的数字方块。画面带有超现实的故障艺术风格，冷色调，背景隐约闪烁着发光的 3D 坐标代码。电影级光影，氛围神秘死寂。）
  - **操作**：与控制台交互出现 3 个槽位。玩家拖入 3 张关联立牌（如钥匙、照片、花）。点击“合成”，调用 Seedance 2.0。银幕上播放 5 秒超现实故障风 MP4，视频最后几帧定格为坐标密码，玩家输入密码锁过关。

- **第五章：重构世界（最终决战）**
  - **视觉**：深不见底的白色虚空，目标是头顶 10 米处发光的逃生门。
  - **机制与判定**：Roguelike 物理堆叠与全档读取。
  - **操作**：读取玩家 `user://inventory.json` 中保存的**所有**立牌数据。由于贴纸生成的是精确的碰撞体（Godot 3D 下通过挤出多边形生成 `CollisionShape3D` 薄片），玩家必须像玩《俄罗斯方块》一样，把这些不规则的物理实体一个接一个拖出并叠高。如果不小心扔了“球形贴纸”，整座塔可能会因为物理引擎的重力模拟而轰然倒塌。

## 4. 存储与持久化

所有提取的物体数据（`name`、`base64_image`、`mass`、`elements`、`tags`、`polygon_points` 等）将持久化存储在客户端的 `user://inventory.json` 文件中，确保玩家能跨进度保留他们精心扫描的“现实物体遗产”。
