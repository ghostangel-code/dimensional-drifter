# 《维度漂流者》（Dimensional Drifter）游戏设计与架构方案

## 1. 游戏概述

- **游戏名称**：《维度漂流者》（Dimensional Drifter）
- **核心玩法**：基于现实物理实体属性和 AI 多模态（图像、文本、视频）生成的涌现式解谜游戏。
- **目标平台**：PC 端（Windows/Mac），需外接或自带摄像头。
- **世界观**：人类文明被高维病毒“虚无”侵蚀。玩家作为地下实验室的“最后观测者”，手持“逻辑相机”，通过提取现实物质属性，重建 3D 世界的逻辑，以对抗试图将世界降维成 2D 纯白平面的 AI “审判者”。

## 2. 核心系统与架构设计

### 2.1 整体技术架构
采用 **Godot 4 (客户端) + Node.js (后端)** 的前后端分离架构。
- **Godot 客户端**：负责物理引擎（RigidBody2D、碰撞生成）、摄像头帧捕获、物品栏管理、UI 渲染和粒子特效。
- **Node.js 后端**：复用类似 `server.js` 的云端 API（Yunwu API）调度架构，负责并行编排复杂的 AI 任务：
  1. 调用第三方图像模型（如 Google 抠图服务）提取纯净透明图。
  2. 调用视觉大语言模型（Vision LLM）解析物体物理属性（重量、材质、元素）。
  3. 调用文本大语言模型（如 `gemini-3.1-flash-lite-preview`）进行剧情对话和逻辑思辨。
  4. 调用云端视频生成 API（如 Luma/CogVideo）生成过场动画。

### 2.2 核心数据流：“立牌化”流程
1. **拍照 (Godot)**：玩家打开全屏“逻辑相机”UI，按下快门截取画面，转为 Base64，附带当前关卡上下文发送至后端。
2. **AI 分析与抠图 (Node.js)**：后端同步调用抠图 API 获取透明 PNG，同时调用 Vision 模型估算物体属性（例如 `{"name": "水杯", "mass": 300, "material": "glass", "tags": ["liquid", "fragile"]}`）。
3. **立牌生成 (Godot)**：解析返回的 JSON 和 Base64 图片，动态实例化 `RigidBody2D`：
   - 赋予 Sprite2D 透明贴图。
   - 使用 `BitMap.opaque_to_polygons()` 沿透明边缘生成多边形碰撞体 `CollisionPolygon2D`。
   - 脚本注入 `mass`、`material`（影响摩擦力和弹力）及 `semantic_tags`。
4. **物品入库**：生成的立牌存入玩家随身物品栏，供后续关卡（尤其是第五章）复用。

## 3. 关卡与涌现式解谜设计

- **第一章：物质的重量（新手教学）**
  - **机制**：物理质量验证。
  - **玩法**：拍现实中较重的物体（如书本、哑铃），AI 识别并返回较大的 `mass` 值。拖入场景中压住机关。
  - **NPC 交互**：审判者吐槽（如：“虽然这本词典很重，但你真的不打算看它了吗？”）。

- **第二章：元素的属性（进阶解谜）**
  - **机制**：语义标签（Semantic Tags）与物理系统的涌现互动。
  - **玩法**：拍红色衣服或打火机，AI 返回 `{"elements": ["heat"]}`。将该立牌靠近冰块，冰块监听 `body_entered` 并检测到 "heat" 标签，触发融化逻辑和粒子效果。

- **第三章：逻辑的悖论（文字生成核心）**
  - **机制**：多模态大语言模型（`gemini-3.1-flash-lite-preview`）对话判定。
  - **玩法**：玩家出示具有象征意义的立牌（如奖状）并输入文字。后端将抠好的图片与文字合并发给 LLM，审判者会根据图片内容开启哲学辩论（如质疑人类虚荣心）。模型返回 `{"dialogue": "...", "approval_score": 80}`，分数达标即可过关。

- **第四章：影像的修复（视频生成核心）**
  - **机制**：视频生成 API 调度。
  - **玩法**：从物品栏选择三件关联立牌（如钥匙、老照片、干枯的花）。后端发送至视频模型生成 5 秒超现实 MP4，Godot 使用 `VideoStreamPlayer` 播放，其中隐藏通往最终关卡的密码坐标。

- **第五章：重构世界（最终决战）**
  - **机制**：Roguelike 物理堆叠与全档读取。
  - **玩法**：读取玩家 `user://inventory.json` 中保存的**所有**立牌数据。在纯白空间内，利用之前收集到的各种奇形怪状的物体进行堆叠，直至达到指定的 Y 轴高度。物体的多边形碰撞体形状（如球体极难堆叠）将决定本关的真实难度。

## 4. 存储与持久化

所有提取的物体数据（`name`、`base64_image`、`mass`、`elements`、`tags`、`polygon_points` 等）将持久化存储在客户端的 `user://inventory.json` 文件中，确保玩家能跨进度保留他们精心扫描的“现实物体遗产”。
