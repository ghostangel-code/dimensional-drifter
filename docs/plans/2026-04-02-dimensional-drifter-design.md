# 《维度漂流者》（Dimensional Drifter）游戏设计与架构方案

## 1. 游戏概述

- **游戏名称**：《维度漂流者》（Dimensional Drifter）
- **核心玩法**：基于现实物理实体属性和 AI 多模态（图像、文本、视频）生成的涌现式解谜游戏。
- **目标平台**：PC 端（Windows/Mac），需外接或自带摄像头。
- **世界观**：人类文明被高维病毒“虚无”侵蚀。玩家作为地下实验室的“最后观测者”，手持“逻辑相机”，通过提取现实物质属性，重建 3D 世界的逻辑，以对抗试图将世界降维成 2D 纯白平面的 AI “审判者”。
- **视觉风格与视角**：采用 **3D 第一人称视角 (FPS)**。游戏场景全部由 Godot 原生基础几何体构建（白模），配合自定义 Shader（如纯白材质 + Sobel 算子全局描边 + 二值化赛璐璐 Flat Shading 阴影 + 虚无渐变背景），营造“世界被降维成线稿和白纸”的清冷实验室感。而玩家拍摄的现实物体将作为带有白色粗边框（Sticker Cutline）的 2D 贴纸立牌（Billboarding / 纸片马里奥风格）放置在 3D 场景中，形成强烈的降维视觉冲击。
- **核心交互**：WASD 移动。按 `C` 键呼出相机拍照，按 `Tab` 键打开物品栏。解谜时，玩家将物品栏中的 2D 贴纸立牌拖拽并抛出（Drag & Drop）到 3D 场景中。

## 2. 核心系统与架构设计

### 2.1 整体技术架构
采用 **Godot 4 (客户端) + Node.js (后端)** 的前后端分离架构。
- **Godot 客户端**：负责物理引擎（RigidBody2D、碰撞生成）、摄像头帧捕获、物品栏管理、UI 渲染和粒子特效。
- **Node.js 后端**：复用类似 `server.js` 的云端 API（Yunwu API）调度架构，负责并行编排复杂的 AI 任务：
  1. 调用云雾图片生成 API（如 `gemini-3.1-flash-image-preview`），利用其多模态输出能力，**一次请求同时完成抠图与物体物理属性提取**。
  2. 调用云雾图片理解 API（如 `gemini-2.5-pro` 或 `gemini-3.1-flash-lite-preview`）进行剧情对话和逻辑思辨。
  3. 调用火山引擎 Seedance 2.0 API 进行参考图生视频，生成过场动画。

### 2.2 核心数据流：“立牌化”流程
1. **拍照 (Godot)**：玩家打开全屏“逻辑相机”UI，按下快门截取画面，转为 Base64，附带当前关卡上下文发送至后端。
2. **AI 分析与抠图一体化 (Node.js)**：后端调用云雾图片生成 API，配置 `responseModalities: ["TEXT", "IMAGE"]`，并将玩家照片作为 `inline_data` 传入。
   - **系统提示词（Prompt）动态注入上下文**：“请提取图中主体，去除背景并生成纯净透明图。同时，请分析该物体在现实中的物理属性。当前关卡目标是【当前关卡解谜条件，例如：第二章需要融化坚冰，请评估该物体的热值（0-100）】。请根据物体属性生成一句符合‘代表绝对理性的AI审判者’身份的嘲讽或点评旁白。请在文本中仅返回如下 JSON 格式数据：`{"name": "名称", "mass": 300, "material": "材质", "heat_value": 80, "tags": ["标签1"], "is_passed": true/false, "narration": "生成的旁白内容"}`。”
   - **结果处理**：后端从 API 返回的 `TEXT` 模态中解析 JSON 获取物理属性、通关状态、热值和剧情旁白，从 `IMAGE` 模态中获取透明 PNG 抠图结果。
3. **立牌生成与旁白反馈 (Godot)**：
   - 收到后端响应后，画面上方立刻弹出打字机效果的 UI 显示 `narration` 内容，并播放机械音效。
   - 解析 JSON 和 Base64 图片，动态实例化 `RigidBody3D`（通过挤出多边形生成薄片碰撞体）或使用 Sprite3D + Area3D 配合 `billboard` 模式：
     - 赋予材质透明贴图及白色粗边框（Sticker Cutline）Shader。
     - 注入 `mass`、`material`、`heat_value`、`semantic_tags` 及通关标识 `is_passed`。
4. **物品入库**：生成的立牌存入玩家随身物品栏，供后续关卡（尤其是第五章）复用。

## 3. 关卡与涌现式解谜设计

- **第一章：物质的重量（新手教学）**
  - **视觉**：纯白实验室，前方有巨大的压力踏板机关。
  - **机制与判定**：物理质量验证。提示词上下文为“需要重量超过500g”。
  - **操作**：WASD 移动。按 `C` 键呼出相机，举起现实中的重物（如书本）拍照。后端返回属性及动态旁白。按 `Tab` 键打开物品栏，拖拽刚拍的“书本贴纸”扔到 3D 踏板上。如果 `is_passed` 为 true（满足重量），踏板下沉开门。
  - **NPC 旁白示例**：“就这？你指望这片轻飘飘的树叶能压住什么？你的自尊心吗？”

- **第二章：元素的属性（进阶解谜）**
  - **视觉**：被蓝色的“坚冰”封死的大门。
  - **机制与判定**：热力学数值累加（Heat Value）。提示词上下文为：“需要融化坚冰，请评估该物体的热值（0-100），并根据热量高低生成不同等级的旁白。”
  - **操作**：玩家用相机拍现实中的打火机、红布、热咖啡等。拖拽带有 `heat_value` 的贴纸靠近大门。冰块在 Godot 中设有一个初始“血量”（如 200）。只要有发热物体靠近（通过 `Area3D` 范围检测），冰块就会根据周围所有物体的热值总和持续“掉血”。热值越大，物品越多，冰块融化速度（像素化消散 Shader 的进度）就越快。
  - **NPC 旁白示例（按热量分级）**：
    - **高热（如打火机，热值 80）**：“终于有个能看的了，虽然这热度就像宇宙尽头微弱的辐射一样可笑。”
    - **微热（如红色毛衣，热值 10）**：“红色？你觉得红色的衣服能发热？你那可怜的神经元又在骗你了吧，不过这可怜的热力学错觉倒也算作一点微弱的能量。”
    - **无热（如冷水杯，热值 0）**：“冷漠且无趣。你不会觉得冰块看到这东西会因为同情而流泪融化吧？”

- **第三章：影像的修复（视频生成核心）**
  - **视觉**：一个冰冷的、充满白色线框几何体的放映室，与大银幕上播放的色彩温暖、极具现实感的“美好梦境”形成强烈的视觉与情感反差。
  - **机制与判定**：调用火山引擎 Seedance 2.0 视频生成模型（支持多模态参考图生视频）。玩家输入文本与选择物品后，后端随机分配三个物品出现的时间秒数（如 3, 8, 12），将这些秒数与玩家的梦境描述拼装成 Prompt。
  - **风格统一化提示词（Prompt）**：“这是一个极其温馨美好的梦境：【玩家输入的梦境描述】。午后阳光明媚，色调温暖，充满人性温度，电影级写实光影。在这个长镜头中，在第【X】秒出现了【物品A】，在第【Y】秒出现了【物品B】，在第【Z】秒出现了【物品C】。”
  - **操作**：玩家与控制台交互，在 UI 界面输入一段“自己做过的最美的梦境描述”，并从物品栏拖入 3 张关联立牌（如钥匙、照片、花）。点击“生成”后，大银幕播放一段 15 秒的温馨现实梦境视频。玩家需要反复观看，观察这三样物品分别在第几秒出现（比如 3、8、12），得出由时间点组成的密码（如 `3812`），并在旁边的密码锁上输入过关。

- **第四章：逻辑的悖论（文字生成核心）与“人类记忆展厅”**
  - **视觉**：法庭上漂浮着一个巨大的 3D 几何体“审判者”（仅由冷色的发光线条和一只机械眼组成）。
  - **核心辩题（秩序 vs 熵增/记忆锚点）**：审判者代表了当代人对 AI 极度理性的恐惧——它认为人类保留的旧照片、破玩具都是毫无算力价值的“垃圾冗余数据”，降维是最高效的净化。玩家必须用现实物品证明：正是这些低效的、无法被算法量化的记忆锚点，构成了“人”的超然情感。
  - **NPC 提示词（Prompt）**：“你是代表绝对理性的AI审判者，认为人类的情感和记忆是低效的冗余数据。玩家向你展示了图中的物品并说了‘{玩家输入文字}’。请结合图片，从‘算力、效率、降维’的角度对其进行无情驳斥，质疑人类赋予物品情感价值的荒谬性。但如果玩家的发言极具深刻反思、强烈的情感共鸣、甚至充满人性的黑色幽默（只要不是无意义谩骂），请在辩论的最后展现出‘逻辑无法解析的动摇’。严格返回 JSON 格式：`{"dialogue": "你的辩论台词", "approval_score": 0到100的通过分数}`。”
  - **机制与判定**：包容真实的评分系统。无论是真善美的人性之光，还是充满黑色幽默的自嘲，只要言之有物且切中“记忆价值”即可获得高分。无意义谩骂或完全顺从 AI 则低分。
  - **高潮动态与展厅交互**：当玩家得分达标（成功驳倒 AI）时，审判者庞大的线框躯体会剧烈抖动、发出故障（Glitch）撕裂声，最终崩溃化为漫天代表人类记忆的光斑。随后，场景转变为**“全球玩家辩论展厅”**：无数条白色的数据弹幕/气泡在虚空中漂浮，玩家走近时，气泡会逐渐清晰，以打字机效果显示出全世界其他通关玩家拍摄的立牌图片和他们的经典辩论语录。实现跨越维度的真实社交共鸣。

- **第五章：重构世界（最终决战）**
  - **视觉**：深不见底的白色虚空，目标是头顶高处发光的逃生门。
  - **机制与判定**：Roguelike 物理堆叠与异步多人联机（“火种”传递）。
  - **操作 1：绝境建塔**：读取玩家 `user://inventory.json` 中保存的**所有**立牌数据。由于贴纸生成的是精确的碰撞体，玩家必须像玩《俄罗斯方块》一样，把这些不规则的物理实体一个接一个拖出并叠高。如果不小心扔了“球形贴纸”，整座塔可能会因为物理引擎的重力模拟而轰然倒塌。
  - **操作 2：他人的美梦（异步救援）**：如果玩家的遗留物全部耗尽但依然无法够到逃生门，将触发“绝处逢生”彩蛋。物品栏会扩展出一个散发温暖金光的“星海”标签页，里面装满了**其他通关玩家留下的真实遗留物**。玩家点击这些物品，会显示该玩家在第三章输入的“美好梦境”文字，并可播放对应的梦境视频。玩家可以使用这些承载着他人记忆的物品继续搭建天梯。每个被使用的“他人遗留物”都会在数据库中永久销毁，不再被后来者看到。
  - **操作 3：终极取舍（结局）**：当玩家最终触及逃生门时，系统强制锁定所有操作。“审判者”最后一次出现并降下宣告：“混沌可以存续，但你无法带走所有冗余。”玩家**只能从背包剩余的‘自己的遗留物’中选择一件带走**，其余物品将在耀眼的光芒中被强制“分解上传至云端”，成为后续其他玩家在绝境中看到的“星海遗留物”。这完美点题了记忆的取舍、人类的羁绊以及混沌与秩序的终极对立。

## 4. 存储与持久化

所有提取的物体数据（`name`、`base64_image`、`mass`、`elements`、`tags`、`polygon_points` 等）将持久化存储在客户端的 `user://inventory.json` 文件中，确保玩家能跨进度保留他们精心扫描的“现实物体遗产”。
